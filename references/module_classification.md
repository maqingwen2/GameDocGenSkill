# 游戏功能模块分类指南

## 标准模块体系

基于常见手游/页游策划案结构，功能模块按层级划分如下：

### L1 系统级模块

| 模块编号 | 模块名称 | 说明 |
|---------|---------|------|
| 1 | 功能解锁 | 活动/功能的开启条件、前置要求 |
| 2 | 活动时间 | 玩法时间、领奖时间、活动周期 |
| 3 | 主界面&通用功能 | 入口界面、通用 UI、通知系统 |
| 4 | 核心玩法 | 该功能的主要游戏玩法 |
| 5 | 辅助功能 | 练习模式、教学、引导 |
| 6 | 商业化 | 战票、通行证、付费相关 |
| 7 | 社交系统 | 排行榜、好友、分享 |
| 8 | 奖励系统 | 任务、成就、签到、掉落 |

### L2 界面级子模块

#### 3. 主界面&通用功能
- 3.1 主界面：背景、标题、主要入口按钮
- 3.2 排行榜：排名列表、自己的排名、排行榜类型切换
- 3.3 排行奖励：根据排名发放的奖励预览
- 3.4 规则说明：玩法规则文本展示
- 3.5 帮助/FAQ：常见问题解答

#### 4. 核心玩法
- 4.1 进入确认：进入玩法前的确认弹窗/条件检查
- 4.2 玩法界面：核心游戏界面
- 4.3 技能/道具系统：可用技能、BUFF、道具
- 4.4 结算界面：单局结束、分数展示、奖励发放
- 4.5 游戏结束：完全结束、中止、通关判定

#### 6. 商业化
- 6.1 活动战票：通行证/战令系统
- 6.2 商城/礼包：直接购买道具
- 6.3 充值入口：货币充值界面

#### 8. 奖励系统
- 8.1 每日任务：日常任务列表及奖励
- 8.2 成就系统：长期目标及奖励
- 8.3 签到系统：连续签到奖励
- 8.4 邮件系统：系统邮件、奖励领取

## 模块识别特征

### 主界面识别
- 存在明显的功能入口按钮（开始游戏、排行榜、规则等）
- 展示玩家基础信息（头像、昵称、等级）
- 有活动主题背景或标题

### 排行榜识别
- 列表形式展示多个玩家条目
- 每个条目包含排名数字、玩家名、分数
- 通常有自己的排名高亮显示
- 可能有时间范围筛选（日榜/周榜/总榜）

### 背包识别
- 网格或列表展示多个物品槽位
- 物品有图标、数量、品质颜色
- 有分类标签（全部/装备/消耗品）
- 可能有整理/出售/使用按钮

### 战斗系统识别
- 实时变化的状态条（HP/MP/能量）
- 可点击的技能/攻击按钮
- 伤害数字飘字
- 敌人/己方的角色模型

### 抽卡系统识别
- 卡池名称和描述
- 单次/十连抽按钮
- 概率说明
- 抽卡结果展示（角色/道具）
- 保底计数器

### 任务系统识别
- 任务名称和描述列表
- 进度显示（3/5）
- 奖励预览图标
- 完成状态标识（已完成/进行中）

## 输出编号规则

生成的策划案中，模块编号遵循以下规则：

1. **一级模块**：阿拉伯数字，如 `1`、`2`、`3`
2. **二级模块**：`一级.二级`，如 `3.1`、`3.2`
3. **三级模块**：`一级.二级.三级`，如 `4.2.1`
4. **详细设计条目**：在模块编号后加小数，如 `3.1.1`、`3.1.2`

编号与模块标题之间用空格分隔，如：
```
3.1 主界面
3.1.1 背景展示
```

## 特殊模块标记

以下情况需要特殊标注：

- **通用功能（跨模块）**：在模块名前加 `[通用]`，如 `[通用]红点提示`
- **配置表关联**：在描述中用 `【TableName】` 标注
- **文本 Key**：在文案后用括号标注 Key 格式，如 `活动标题 (EventName_Ui_Title_001)`
- **待确认项**：用 `【需补充】` 前缀标注
